## 쓰래싱(Thrashing)

> - 어느 수준 이하에서  
> 멀티프로그래밍도가 증가하면 CPU 활성도는 덩달아 늘어난다.  
  
> - 어느 임계점을 넘어서는
> 멀티프로그래밍이 어느 수준 이상으로 증가하면 실제 유효한 작업을 하지 못하고 페이지 폴트만 처리한다.
> CPU 활성도는 곤두박질치고, 이것을 쓰래싱이라 부른다.  
  
> - 임계점이란?  
> Locality의 총합 < 실제 물리 메모리(m)일 때.
  
## Working-Set Model  
  
> 지역성을 추정하여 필요한 만큼의 메모리 프레임을 할당하는 기법.  
> 워킹셋 창(델타)이 Working Set Size(D)보다 작으면 쓰래싱 발생.
> 따라서 메모리 요구량을 실시간으로  파악하며 워킹셋 창의 크기를 동적으로 변경해, 쓰래싱이 발생하지 않도록 함.
